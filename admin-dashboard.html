LingModel upstream + tier mapping

Which provider URL receives proxied traffic, and which upstream model id each tier resolves to. Empty visible field = fall back to env (deploy-time default). No restart required.

Save first, then Test to confirm the live upstream (URL + key + default model) actually responds.

Image generation upstream

Used by /api/images/generate. Defaults to OpenAI's images/generations; key falls back to the OPENAI_API_KEY env var if no DB key is set.

LingModel rate limits

Per-tier daily prompt cap, 5h-window prompt cap (Pro only), daily/monthly output token caps, and free-tier burst limit (req/min). Each field is a non-negative integer; 0 means "no cap" (operator escape hatch). Empty = clear DB override → fall back to env, then to the hard-coded default. Effective immediately, no restart required.

Image generation (monthly caps)

Per-tier monthly image count. 0 = no cap. Resets on the 1st (UTC). Defaults are intentionally low on free since gpt-image-1 runs $0.04–0.17 per image.

Cloud backend limits (per tier)

Per-tier caps for LingCode Cloud backends: database tables, storage objects + max object size, auth users, cloud functions + per-invocation timeout, and managed emails/day. Each field is a non-negative integer. Empty = clear the DB override → fall back to the built-in default. Effective immediately, no restart required.

Deep Agent (server-side Claude SDK)

Pro/Max-only agent surface in /try. Token caps are monthly per user; usage resets on the 1st. Kill switch overrides everything.
