在 Vultr 上购买服务器

Vultr 的核心价值在于地理覆盖。30 多个数据中心分布在六大洲，每处均提供形态统一的小型 VPS。如果你的用户遍布全球，单靠"纽约数据中心"或"法兰克福数据中心"难以全面覆盖，Vultr 正是为此而生。

VPS 市场的竞争主要集中在三个维度：价格、易用性和节点位置。DigitalOcean 在易用性上领先，价格属于市场正常水平；Hetzner 价格最低，易用性也不差；而 Vultr 的定位是"节点比两者都多，价格比 Hetzner 贵、比 DigitalOcean 便宜"。如果地理位置对你很重要——比如你的用户同时分布在首尔、孟买、圣保罗、悉尼，不想让他们所有人都连接到地球另一端的服务器——Vultr 的节点覆盖就是它最大的差异化优势。

具体来说：Vultr 在东京、首尔、新加坡、悉尼、墨尔本、孟买、班加罗尔、特拉维夫、斯德哥尔摩、马德里、圣保罗、墨西哥城、约翰内斯堡等城市都有数据中心。这些地点 AWS 部分没有，DigitalOcean 全部没有，而 Hetzner 仅有的五个节点更是一个都不覆盖。对于小型全球化产品来说，"在对的城市有小型 VPS"有时比"在地球另一端有强大的 VPS"更有价值。

本教程将介绍 Vultr 的购买流程——它与 DigitalOcean 的操作大同小异，大部分步骤可以直接复用——同时着重说明两者的不同之处：Marketplace 镜像、按小时计费的细节，以及什么情况下选择 Vultr 比另外两家更合适。

你将学到

  • Vultr 的套餐体系：Regular、High-Performance、High-Frequency
  • 根据目标用户选择数据中心
  • SSH 密钥配置与服务器创建
  • 通过 Vultr Marketplace 一键安装应用
  • 什么时候 DigitalOcean 或 Hetzner 是更好的选择

第 1 步：选择套餐系列

1

三个档次，通常选中间那个

Vultr 的套餐系列：

  • Cloud Compute（普通性能）：Intel CPU。入门价格最低，$2.50/月起配 512MB 内存。$2.50 的配置确实很小，但能用；$5–6 可以获得 1GB，足以跑许多小型服务。
  • High-Performance（HP）：采用更新的 AMD EPYC CPU，同价位性能优于 Regular。优先选这个档次（如果你所在区域有的话）——这是 Vultr 竞争力最强的套餐。
  • High-Frequency（HF）：针对单线程性能优化（3.8+ GHz）。适合对单线程速度敏感的场景（老旧 PHP 应用、某些数据库等）。比较小众。

如果有需要，也可以选择 Cloud GPU 套餐。第一台服务器先跳过。

第 2 步：选择区域

2

选 Vultr 的核心理由

Vultr 会列出每个区域的当前容量和功能可用情况。其他服务商没有而 Vultr 有的节点：

  • 首尔、东京、大阪——覆盖日本/韩国。
  • 孟买、班加罗尔、德里——三个印度城市（Hetzner 一个都没有）。
  • 悉尼、墨尔本——覆盖澳大利亚。
  • 圣保罗、墨西哥城——拉丁美洲（美国云服务商通常只有圣保罗）。
  • 约翰内斯堡——本指南中唯一一个非洲数据中心。

如果你的用户在上述任何市场，Vultr 往往是独立 VPS 层面唯一合适的选择。价格比 AWS 同区域便宜，距离用户又远比 DigitalOcean 近。

第 3 步：配置 SSH 密钥、选择镜像、创建服务器

3

标准 VPS 创建流程

在 Account → SSH Keys 中添加你的 SSH 公钥（和你在其他 VPS 上使用的 Mac 密钥相同）。然后点击 Deploy New Server：

  • 服务器类型：Cloud Compute（如果有，选 HP 档次）。
  • 位置：按第 2 步选择。
  • 镜像：Ubuntu 24.04 LTS，或从 Marketplace 镜像中选一个预装好的技术栈（Docker、WordPress 等）。
  • 套餐：选你需要的最小配置——以后可以升配（降配则比较麻烦）。
  • SSH Keys：选刚才添加的那个。
  • 主机名：起一个好记的名字。

点击 Deploy Now。服务器通常在 30–60 秒内完成创建。

Vultr Marketplace：镜像选择器里有一个"Marketplace Apps"标签页，提供预构建镜像（Docker、WordPress、GitLab、Plex 等）。如果你想直接用现成的技术栈而不想从头安装，这个功能很方便。这些镜像由社区维护——用于生产环境之前，务必查看镜像的最后更新日期。

第 4 步：SSH 登录并加固服务器

4

与其他 VPS 操作相同

初始十分钟的加固流程与 DigitalOcean 或 Hetzner 完全一致：

ssh root@<server-ip>
apt update && apt upgrade -y
adduser deploy
usermod -aG sudo deploy
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy
ufw allow OpenSSH && ufw enable
# 确认 deploy 用户可以正常登录后：禁用 root SSH 登录

操作系统是标准的 Ubuntu，所有针对 Ubuntu 的 Linux 教程都可以直接使用，无需任何适配。

第 5 步：计费方式——按小时，但要注意细节

5

"删除服务器不等于停止计费"的陷阱

Vultr 和 AWS 一样按小时计费。删除服务器会停止计算费用，但要注意：

  • 单独存储的快照在源服务器删除后仍会持续产生存储费用。请手动删除不需要的快照。
  • 保留 IP 地址每月约 $3——如果你分配了一个保留 IP 并在未释放的情况下解绑，费用会持续计算。
  • 备份快照（如果开启了自动备份选项）会按独立的计划持续计费。

建议每月查看一次 Billing 页面。Vultr 会清晰地展示未关联的孤立资源——但前提是你要主动去看。

第 6 步：什么情况下不选 Vultr

6

两种真实情况

  • 你不需要特殊的地理覆盖。如果你的用户主要在北美或西欧，DigitalOcean 的易用性或 Hetzner 的价格各有优势，Vultr"介于两者之间"的定位反而没有明显吸引力。
  • 你需要完善的文档和活跃的教程生态。Vultr 的文档还不错，但 DigitalOcean 的明显更好。Stack Overflow 上关于 DO 的社区资源也更丰富。如果你会花大量时间查资料，这个生态差距不可忽视。

第 7 步：绑定域名

7

标准 A 记录

在你的 DNS 服务商处添加一条 A 记录，名称填 @ 或 app，值填你的 Vultr 服务器 IPv4 地址。详见 将域名连接到服务器。几分钟后，域名即可指向 Vultr。

部分小众功能需要额外付费。IPv6 是免费的，但自定义 ISO 上传、BGP 会话和保留 IP 都会产生小额月费，这些费用不包含在套餐标价中。如果你开始使用这些功能，记得在账单出来前提前确认费用。
