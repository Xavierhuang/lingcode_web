在 Linode（Akamai Cloud）购买服务器

Linode 是那个教会一代开发者租用 Linux 服务器的 VPS 服务商。创立于 2003 年，2022 年被 Akamai 收购，如今以"Akamai Cloud Computing"的名义运营。底层产品基本未变——稳定可靠，价格略高于 DO 和 Hetzner——但品牌形象仍处于过渡期。

在 DigitalOcean 出现之前，在按小时计费的独立云市场诞生之前，就有了 Linode。他们自 2003 年起就在出租虚拟 Linux 服务器，比现代 VPS 市场的大多数玩家都早，并以"对开发者友好、技术扎实"的形象在企业级主机市场独树一帜。Linode 的社区围绕其论坛形成；他们的教程让无数自学成才的 Web 开发者学会了 Linux 运维；在一个诚信并不常见的时代，他们的定价保持了难得的透明。

2022 年，Akamai 收购了 Linode。产品依然在，用户体验也基本保住了，但品牌正逐渐被纳入 Akamai Cloud 的企业定位之中。新用户今天看到的是"Akamai Cloud Computing"，"Linode"退居副品牌。定价结构没有变化，控制台基本还是原来的 Linode 风格，只是慢慢融入了一些 Akamai 的整合功能。

对今天的开发者来说，Linode 大致相当于"DigitalOcean 的兄长版——价格略贵，企业感略强，但骨子里还是同一种独立云的气质"。坚持用它的理由是情怀——如果你已经用了十年，没有理由离开。放弃它的理由也很实在：DigitalOcean 在文档和上手体验上已经超过它，Hetzner 在价格上则把它甩开一大截。Linode 的位置是"第三名"，而在这个市场里，第三名没有太强的存在感。

你将学到什么

  • 收购前后有什么相同点，有什么不同点
  • Linode 的套餐结构（与 DO 类似）
  • 创建一台 Linode 并通过 SSH 连接
  • Linode 文档库——平台上依然最好的部分
  • 什么时候应该选择其他 VPS

第 1 步：注册账号

1

cloud.linode.com

在 cloud.linode.com 注册账号。注册流程简单直接；在创建任何实例之前需要绑定支付方式。新账号通常可获得小额免费额度（100 美元，有效期 60 天），作为入驻奖励——Linode 会定期推出这类优惠活动。

你也会看到"Akamai Cloud"的字样——产品相同，品牌不同，不必困惑。cloud.linode.com 才是官方入口。

第 2 步：选择套餐

2

共享 CPU 是入门层级

Linode 的套餐系列：

  • Nanode 1GB——入门层，5 美元/月，1 vCPU / 1 GB 内存 / 25 GB SSD。与 DO 的 5 美元 Droplet 相当。
  • Linode 2GB / 4GB / 8GB——共享 CPU，逐级扩容。与 DO 基础套餐结构和定价相同。
  • 独享 CPU——保证无"嘈杂邻居"干扰的实例，价格约为共享套餐的 2 倍。
  • 高内存、GPU——面向特定需求的专业层级。

共享 CPU 套餐足以覆盖绝大多数使用场景。更大的负载选更高的套餐；入门层级完全够用且性能真实。

第 3 步：创建 Linode

3

与任何 VPS 的操作流程相同

点击 Create → Linode，填写以下信息：

  • 发行版：Ubuntu 24.04 LTS。
  • 地区：选择离用户最近的节点。Linode 拥有 20 多个区域，北美和欧洲覆盖良好，亚洲也有一定覆盖。
  • 套餐：参见第 2 步。
  • 标签：你的服务器名称。
  • Root 密码：必填，即使你之后只使用 SSH 密钥登录。生成一个长密码后可以丢弃。
  • SSH 密钥：粘贴你的公钥（~/.ssh/id_ed25519.pub）。这才是你真正使用的凭证。

点击 Create Linode。配置需要 1–2 分钟（比 DO/Hetzner 稍慢）。完成后，公网 IPv4 地址将显示在控制台中。

第 4 步：SSH 登录并加固服务器

4

与其他地方的流程完全相同

初始安全加固步骤与其他 VPS 完全一致：

ssh root@<linode-ip>
apt update && apt upgrade -y
adduser deploy
usermod -aG sudo deploy
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy
ufw allow OpenSSH && ufw enable
# 确认 deploy 用户可正常登录后：编辑 sshd_config，将 PermitRootLogin 设为 no

Linode 还内置了云防火墙产品（对 Linode 用户免费），运行在网络边缘、流量到达虚拟机之前。将其与 ufw 配合使用，实现纵深防御。

Linode 文档库。即便在收购之后，linode.com/docs 的教程库依然是网络上最好的免费 Linux 管理资源之一。数百篇教程，写作认真，持续更新。即使你不使用 Linode，这些文档也值得一读。（DigitalOcean 的文档同样出色，两者堪称独立云文档的黄金标准。）

第 5 步：备份与快照

5

两种相关产品

Linode 提供两种类似备份的功能：

  • 备份（Backups）——每台 Linode 的付费附加功能（约 2–5 美元/月，视套餐而定）。自动执行每日/每周/双周快照并按轮换策略保留。适合希望"设置好就不管"的数据保护需求。
  • 快照（Snapshots）——按需创建的时间点镜像，拍摄免费，但存储期间计费。适合"在这次风险操作之前留个备份"的场景。

对于非关键项目，在高风险操作前打快照就足够了。对于任何你不想丢失的内容，每月花一杯咖啡的钱开启备份功能是值得的。

第 6 步：什么时候该换其他服务商

6

说实话：通常有更好的选择

  • 体验更好。DigitalOcean 在界面和上手体验上已超过 Linode。如果你想要最友好的独立云，选 DO。
  • 价格更低。Hetzner 同等配置要便宜得多。如果性价比是首要考量，选 Hetzner。
  • 区域更广。Vultr 在亚洲和拉美有更多节点。如果你需要某个特定的偏远区域，看看 Vultr。
  • Akamai 生态整合。如果你已经在使用 Akamai 的 CDN 或其他服务，Linode 与 Akamai 平台日益深化的整合确实是个加分项。这是"被收购"唯一真正有利的地方。

2026 年选择 Linode 的真实理由：如果你已经在用它且用得顺手，就继续用。不要无缘无故从别的地方迁移过来。

第 7 步：绑定域名

7

标准 A 记录

在你的 DNS 服务商处添加 A 记录：名称填 @ 或 app，值填你的 Linode IPv4 地址。参见将域名连接到服务器。

Linode 也有自己的 DNS 管理器（免费，独立于 Linode 实例）。如果你希望 DNS 和计算资源在同一个控制台管理，可以将域名的 DNS 服务器委托给 Linode——不过注册商自带的免费 DNS 或 Cloudflare 的免费 DNS 同样好用。

Akamai 品牌重塑是个渐进过程。有些页面写"Akamai Cloud"，有些写"Linode"，有些两者都有。产品是同一个；这种不一致纯属外观问题，但偶尔会让人一时摸不着头脑。文档页面可能在 linode.com，也可能在 techdocs.akamai.com，具体取决于文章的更新时间。
