从 Cloudflare Registrar 购买域名

Cloudflare Registrar 按注册局原价收费，无加价、无附加销售、无首年低价续费高价陷阱。唯一的限制是你必须使用 Cloudflare 的 DNS。对大多数开发者项目来说，这其实是一个优点，而非约束。

大多数域名注册商同时做两门生意。第一是枯燥的数据库业务——记录谁拥有哪个域名。第二才是真正赚钱的地方：向你推销附加服务——邮箱、SSL、"高级" DNS、托管、隐私保护。结账时 $9 的域名常常变成 $50 的附加包，第一年 $9 到第二年就变成 $18。这种摩擦很烦人——你得花时间取消勾选各种选项——而定价本身也让那些只想买个名字、然后继续工作的人感到不爽。

Cloudflare Registrar 的卖点是他们只做第一门生意。他们按注册局批发价收费（撰写本文时 .com 约为 $9.15），原价转给你，不加一分钱。没有"续费加价"，结账流程里也没有附加推销。价格就是价格，年年如此。

代价是：Cloudflare 要求域名使用 Cloudflare 的域名服务器。你不能把域名放在 Cloudflare 却用别家的 DNS；域名必须全程托管在你的 Cloudflare 账户中。对大多数开发者来说这完全没问题——Cloudflare 的 DNS 又快又免费，各类记录类型一应俱全。如果你坚持使用其他 DNS 提供商，那这就是个死局。这就是全部的限制条件。

你将学到

  • 为什么 Cloudflare 能比 Namecheap 收费更低（这不是亏本引流）
  • 哪些域名可以在这里注册，哪些不行
  • 一步完成域名注册的流程
  • "使用 Cloudflare 域名服务器"为何大多数情况下是优点
  • 什么情况下应该选择其他注册商

第 1 步：创建 Cloudflare 账户

1

免费，一分钟搞定

在 cloudflare.com 注册账户。免费套餐已足够完成本教程的所有操作。记得开启双重验证（2FA）——这个账户将同时管理你的域名和 DNS，值得多一道防线。

如果你已经因为 Cloudflare 其他产品（Workers、R2、Pages、Tunnel 等）注册过账户，直接用就好，无需单独注册注册商账户。

第 2 步：搜索域名

2

控制台 → Domain Registration → Register Domains

在 Cloudflare 控制台中找到 Domain Registration → Register Domains。Cloudflare 时常调整控制台导航结构；如果截图里的位置已经变了，用控制台顶部的全局搜索（/）能立即找到——输入"register"即可。

搜索你想要的域名。Cloudflare 会提前显示每个它支持的顶级域名的价格——重要的是，这个价格年年不变，不存在"第一年 $9，续费 $18"的套路。

并非所有顶级域名都支持——Cloudflare 覆盖的是一个子集，聚焦于开发者常用的扩展名（.com、.net、.org、.io、.dev、.app、.ai、各国家代码等）。如果你想要的顶级域名不在列表中，就需要去别处注册。

比较价格：Cloudflare 显示的数字是注册局批发价加 ICANN 费用。.com 目前约为 $9.15。对比时要看其他注册商的续费年价格（而非首年促销价），差距通常在每年 $5–10 之间。

第 3 步：结账——没有任何东西需要跳过

3

一个项目，一次点击

购物车里只有域名本身。没有 SSL 推销，没有高级 DNS 推销，没有建站工具，没有邮箱。WHOIS 隐私保护默认免费开启。

填写联系信息（用于注册局登记，公开 WHOIS 中会被隐私保护遮蔽），输入支付方式，确认。域名就是你的了。

第 4 步："必须用 Cloudflare 域名服务器"这件事

4

为什么这大多数情况下是优点

结账完成后，Cloudflare 会立即为你的域名创建一个 DNS 区域，并分配他们的域名服务器（NSx.cloudflare.com，两个）。你无法更换域名服务器，只能在 Cloudflare 内部管理 DNS 记录。

对大多数开发者项目而言，这是好事：

  • Cloudflare 的 DNS 是全球速度最快的之一。免费、任播、缓存完善。
  • 控制台界面简洁。A、CNAME、MX、TXT 记录一应俱全，界面现代，没有奇怪的限制。
  • 通过 Cloudflare CDN 免费获得 TLS。如果你把域名通过 Cloudflare 的代理指向某台服务器，HTTPS 会自动启用。
  • 免费套餐包含 DDoS 防护。你可能用不上，但它就在那里。

以下情况则是真正的限制：你的公司因现有基础设施或合同要求，必须使用第三方 DNS 提供商（Route 53、DNSimple 等）。这种情况下 Cloudflare Registrar 不适合你，应从允许自定义域名服务器的注册商购买，比如 Namecheap 或 Porkbun。

第 5 步：添加第一条 DNS 记录

5

将域名指向某处

域名已注册，但还没有指向任何地方。在 Cloudflare 控制台中点击进入该域名，然后在侧边栏选择 DNS，添加你需要的记录：

  • 将根域名指向某个 IP：添加一条 A 记录，名称填 @，值填服务器 IP。
  • 将 www 指向同一位置：添加一条 CNAME 记录，名称填 www，值填根域名（或再添加一条 A 记录）。
  • 指向 Vercel 或 Netlify 等托管平台：通常添加一条 CNAME 记录，名称填 @（或顶级域；Cloudflare 会自动展平），值填平台提供的目标地址。

DNS 连接教程对各类记录类型有更深入的介绍。简而言之：A 和 CNAME 覆盖了你 95% 的使用场景。

第 6 步：Cloudflare 不是最佳选择的情况

6

三个合理的理由让你选择其他地方

  • 你需要 Cloudflare 不支持的顶级域名。支持列表相当全面，但并不完整。.co.uk 在 Cloudflare？不行。.xyz？可以。决定之前先查一下注册页面。
  • 你绑定了其他 DNS 提供商。如果你的公司在 Route 53 或 NS1 上管理 DNS，就无法从 Cloudflare 迁移过去。请从允许自定义域名服务器的注册商购买，比如 Namecheap、Porkbun 等。
  • 你想把域名和邮箱账单合并到一家。Cloudflare 不卖邮件托管服务。如果你希望一站式搞定，Namecheap 这样的注册商或 Fastmail 这样的邮件服务商（既提供邮箱又可以注册域名）是更好的选择。
  • 你需要在锁定账户时有人工客服可以拨打。Cloudflare 免费套餐没有人工支持——账户恢复完全依赖邮件和双重验证。如果两者都丢失，找回域名会非常困难。对于承载重要品牌价值或未来收益的域名，即便每年费用更高，选择提供付费电话支持的注册商可能是更安全的做法。
想把现有域名转入？Cloudflare 接受从其他注册商转入（需要转移授权码，等待 5–7 天，流程和其他地方一样）。如果域名目前正在使用，建议在开始转移前几天将现有 DNS 记录的 TTL 降至 300——这样域名服务器变更后传播更快，能把切换窗口降到最低。
