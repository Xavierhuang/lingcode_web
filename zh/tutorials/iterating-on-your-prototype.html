迭代你的原型

三种技巧，让你无需从头开始，即可对作品进行打磨、调整和改进。

为什么迭代优于重新开始

打磨作品最快的方法，是在已有基础上继续构建，而不是推倒重来。每次迭代只专注于一个改进点，成本更低，也能保留已经运作良好的部分。

技巧一：使用"继续"进行 AI 细化

1

每次只做一个小改动

第一次构建完成后，点击"继续"来请求调整。示例：

  • "把标题做大一些"
  • "将强调色改为蓝色"
  • "增加各区块之间的间距"
  • "把页脚 logo 移到左侧"
  • "添加深色模式切换"

每次"继续"提示都在上一次输出的基础上构建。AI 会保留已经良好的部分，只修改你要求的内容。

成本效益："继续"通常比全新提示节省 20–30% 的成本，因为 AI 只需修改一小部分，而不是从头重新生成。
2

叠加多次细化

你可以将多次"继续"请求串联起来：

  1. 生成初始版本
  2. 继续："让颜色更鲜艳"
  3. 继续："在功能区下方添加用户评价"
  4. 继续："将 CTA 按钮做大并改为红色"
  5. 满意后导出

每一步都在不重新开始的情况下增添精致感。这比一次性提出所有要求要快得多。

技巧二：手动编辑 HTML

1

直接编辑代码

对于快速修复，你可以自己编辑 HTML 或 CSS：

  1. 点击操作台中的"代码"标签
  2. 找到你想修改的区块（使用 Ctrl+F 搜索）
  3. 直接编辑（修改文本、调整 CSS、重新排列元素）
  4. 实时查看预览更新

这种方式非常适合：

  • 修改文本或文案
  • 调整 CSS 数值（颜色、间距、尺寸）
  • 重新排序或隐藏元素
  • 添加你自己的品牌标识或特定细节
2

结合 AI 与手动编辑

最佳工作流通常是两者混合使用：

  1. 用 AI 生成结构（对布局来说通常更快）
  2. 手动编辑文案、颜色和公司专属内容
  3. 如需结构性调整，再使用"继续"

这样你就能在难点（布局设计）上享受 AI 的速度，在细节（你的品牌）上获得手动编辑的精准度。

技巧三：分版本对比

1

生成变体进行对比

面对重大决策时，可以生成两个版本：

  1. 生成版本 A："浅色主题，圆角设计"
  2. 下载并保存版本 A
  3. 回到操作台（或清空并重新开始）
  4. 生成版本 B："深色主题，直角设计"
  5. 在浏览器标签中同时打开两个版本，并排对比
  6. 选择你偏好的版本，继续迭代

何时迭代，何时重新开始

使用"继续"（迭代）的时机：

  • 整体结构/布局良好，只是颜色或间距需要调整
  • 想再添加一个区块或功能
  • 对设计满意，但想做一处视觉微调

从头开始（新提示）的时机：

  • 布局完全偏离了你的设想
  • 想要一个本质上截然不同的设计（深色 vs. 浅色、简约 vs. 张扬）
  • 意识到需要用完全不同的方式来描述需求
决策法则：如果你能用一句话描述这个改动，就用"继续"。如果需要一段话来说明，就重新开始。

加速迭代的技巧

  • 每次"继续"只改一件事："把标题做大"，而不是"把标题做大、加上评价、再改页脚"。
  • 明确说明位置："让 hero 区块更高"，而不是"让它大一点"（"大"可能指标题、间距、图片等各种东西）。
  • 用 CSS 快速测试：想看看换个颜色是什么效果？直接编辑 CSS，不用等 AI。
  • 保存版本：生成了满意的效果就及时下载。如果下一次迭代让你失望，还可以回退。
  • 批量编辑：如果要做 3 处以上的手动修改，一次性全部完成，不要每改一处就等一次预览。

常见迭代模式

模式一：精炼循环

  1. 生成初始版本
  2. 继续：调整颜色
  3. 继续：调整间距
  4. 继续：补充缺失内容
  5. 手动编辑：填入你的具体文字
  6. 导出

模式二：AI + 手动混合

  1. 生成结构
  2. 手动编辑所有文案/品牌内容
  3. 继续：如有需要，进行最终设计润色
  4. 导出

模式三：探索变体

  1. 生成版本 A
  2. 保存版本 A
  3. 新提示：版本 B（不同风格）
  4. 在浏览器中对比两个版本
  5. 选出胜出版本，继续迭代
