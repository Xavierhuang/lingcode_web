LingCode 是否具备 Hermes 风格的自我进化智能体？

简而言之：是的。LingCode 实现了与 Nous Research 的 Hermes Agent 相同的学习循环——持久记忆、智能体自主提案技能和历史会话全文检索——但运行在原生 macOS IDE 内，支持多种 AI 提供商，并加入了人工审核环节，让你始终掌握主导权。

大多数 AI 编程智能体都是一次性的：完成任务后便什么都忘记了。Hermes Agent 在 2026 年引发广泛关注，正是因为它反其道而行——它会持续学习。LingCode 将同样的循环机制带入其内置智能体，让每一次会话都能降低下一次的成本，而无需将你的代码或记忆上传到他人的服务器。

Hermes 学习循环是什么

Hermes 将自身描述为"唯一内置学习循环的智能体——它从经验中创建技能，在使用中持续改进，主动提示自己持久化知识，并检索自身过往对话。"这个循环建立在三大支柱之上：

  • 持久记忆——智能体能记住用户偏好、决策和反复出现的问题。
  • 技能创建——将重复操作转化为可复用的技能。
  • 会话召回——检索自身历史记录，而不是每次从零开始。

LingCode 如何原生实现相同的循环

LingCode 没有 fork Hermes，而是将同样的三大支柱作为一等公民的本地工具，直接内置到其智能体中。

1. 持久记忆与自我提示

智能体配备了 memory_save 工具，可将持久化笔记写入每个项目的记忆文件和跨项目的用户记忆文件。它沿用了 Hermes 的自我提示机制：在可配置的轮次内（默认 8 轮）如果没有保存记忆，LingCode 会提醒智能体将值得记录的内容持久化——例如用户偏好、非显而易见的项目决策或反复出现的痛点。你可以在设置中调整或关闭这一频率。

2. 技能创建——加入人工审核关卡

当智能体发现自己多次执行了同一套多步骤流程时，可以调用 skill_propose 起草一个新的可复用技能（即斜杠命令）。这正是 Hermes 的"从经验中创建技能"——但有一个刻意的差异：在 LingCode 中，新技能会以草稿形式呈现，由你审核并确认后方可激活。你获得了不断积累的能力，同时避免了智能体悄悄改写自身行为的风险。

3. 会话召回与全文检索

LingCode 对你的历史智能体会话建立索引，并通过基于 SQLite FTS5 全文检索的 session_search 工具对外暴露。你可以询问"上周遇到的那个 bug"或"这个问题我在另一个项目里修过吗"，智能体会检索自己的历史记录——返回匹配片段、会话 ID 及各会话所属项目——而不是凭空猜测。

不只限于 Claude

记忆、技能提案和会话工具都内置于 LingCode 的智能体循环中——因此当你切换提供商（DeepSeek 或任何兼容 OpenAI 的模型）时，学习循环依然随行。多提供商 IDE 的意义就在于：能力不应取决于你今天选择了哪个模型。

自我进化智能体为何重要

一个会遗忘的智能体，意味着你每天早上都要重新给它讲解你的代码库。学习循环从根本上改变了这一成本结构：记忆捕捉项目背后的"为什么"，提案技能固化可重复工作流程的"怎么做"，而会话检索则确保机构知识不会随着对话结束而烟消云散。

常见问题

LingCode 使用的是 Hermes Agent 的代码库吗？
不是。LingCode 独立实现了相同的学习循环概念作为原生工具，其技能格式与 Hermes 使用的 agentskills.io 约定兼容——因此技能可以互通。

LingCode 将智能体记忆存储在哪里？
存储在你本地机器上的每个项目记忆文件和跨项目用户记忆文件中，通过专用的 memory_save 工具写入，IDE 会显示保存操作并应用大小限制。

智能体创建的技能会自动运行吗？
不会。skill_propose 会生成一份草稿，需要你审核并确认——这是刻意设计的安全步骤。

网页版也支持学习循环吗？
是的——在 Claude Code Native 标签页和网页标签页中均可使用，并支持 LingCode 的其他提供商。
