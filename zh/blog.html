指南

如何使用 LingCode 技能

技能是用于运行 AI 工作流的斜杠命令。在聊天输入框中输入 / 即可查看全部命令。

快速上手

打开 LingCode，打开一个项目文件夹，看看底部的聊天输入栏。占位符写着 "Plan, @ for context, / for commands"——这就是使用技能的地方。完整的产品形态见 功能列表；多 Claude Code 会话 见对应指南。

步骤 1：输入 /

输入一个正斜杠 /（后面不要加空格）。输入框上方会弹出包含所有可用命令的面板，按类别分组：Code、Git、Testing、Meta 等。

步骤 2：选择技能

用方向键移动，或直接输入关键词过滤（如输入 rev 快速找到 /review）。按回车或点击即可选中。最近使用的 5 个技能会显示在顶部。

步骤 3：发送

按回车运行技能。AI 会收到该技能的指令、当前文件、选区以及项目上下文。

代码技能

当你打开文件或选中代码时使用。帮助你理解、改进和维护代码。

命令作用
/review审查代码的 Bug、安全、性能、代码风格等问题
/explain解释选中的代码或当前文件
/optimize建议性能优化
/refactor建议重构方向
/fix修复 Lint 与编译错误
/doc为代码生成文档
/rename为变量和函数建议更好的命名
/simplify降低复杂度，提高可读性
/insights分析代码库的模式、技术债、架构
/pr-comments为改动的代码生成 PR 审查评论
/batch在代码库中协调大规模变更

Git 技能

在聊天里直接运行。无需选中文件；它们基于你的 Git 状态工作。

命令作用
/commit用 AI 生成的提交信息创建 Git commit
/push推送到远程，带安全检查
/pr用 AI 生成的描述创建 Pull Request

测试与文档

命令作用
/test为当前文件生成单元测试
/coverage分析测试覆盖率并建议缺失的测试
/readme生成或更新项目 README
/changelog从近期 Git 历史生成 CHANGELOG
/release-notes生成特定版本的发布说明

安全与调试

命令作用
/debug协助调试当前代码中的问题
/security审计代码的安全问题并建议修复
/security-review聚焦 PR 改动的安全审查
/deps审查依赖、建议升级、检查漏洞

元命令与工具

用于配置和扩展 LingCode 的技能。兼容 Cursor 的技能格式。

命令作用
/create-skill创建新 Agent 技能的向导 (SKILL.md)
/create-rule创建 Cursor 规则 (.cursor/rules/*.mdc)
/create-subagent为特定任务创建自定义子 Agent
/migrate-to-skills将规则和命令转换为 skills 格式
/shell将剩余输入作为字面 Shell 命令执行
/update-cursor-settings修改 settings.json 中的编辑器设置
/compact总结对话以释放上下文

示例：/explain

打开一个文件，选中一段代码（或 Cmd+A 全选），输入 /，选择 /explain，按回车。AI 会用朴素的语言解释这段代码。

示例：/commit

修改若干文件后，在聊天中输入 /commit。也可以附带提示：/commit 修复登录 Bug。AI 会分析你的 diff 并以 Conventional Commits 规范创建提交。

小贴士

  • 弹窗没出现？ 输入 / 后不要加空格。弹窗只在输入以 / 开头且尚无空格时显示。
  • 键盘： 上下方向键移动、回车选中、Esc 关闭。
  • Agent 模式： 技能在主聊天和 Agent 模式中都可用。选择技能后启动任务即可。
  • 自定义技能： 通过设置添加，或把 SKILL.md 文件放到 ~/.cursor/skills/ 或 .cursor/skills/。
